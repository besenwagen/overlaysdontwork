Image of Text

Explanation

This example shows an image with text inside of it. I like this example because in some cases it is actually possible to determine if an image has text in it. It is also sometimes possible to determine what the text is. Unfortunately doing so accurately is heavily dependent on:

  1. The quality of the image
  2. The legibility of the font in use
  3. Having clear boundaries between the letters and the other stuff in the image
  4. Having the ability to also understand the structure of the content being displayed.

Chances are, no overlay vendor is going to have the capability to accurately fix text-in-images, as it is a huge hassle.

  • Doing so in the browser will also mean leveraging the user's CPU to do that work - something customers would really not want.
  • A server-side process could do this, but:
    • it would become a horse race between the service and the user, as the generated alternative text would need to be put into place, or
    • the generated text alternative could be cached by the service but it is still another AJAX call that the site owner probably doesn't want
  • Regardless of approach, the challenges of accuracy will always exist.
