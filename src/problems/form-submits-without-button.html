Form submits without button

Search: 🔎

Explanation

One of the more weird things we see on the Web are all sorts of elements being used as buttons that aren't actual HTML buttons. Instead, developers are in the practice of creating them out of links, SPANs, and DIVs. Without the necessary roles, states, and properties in place for such pseudo-buttons, they can be pretty problematic for keyboard-only users and screenreader users. The fix for such a thing is easy: use an actual button. To do so an overlay vendor would need to:

  1. Detect objects on the screen that have events bound to them
  2. Replace those elements with buttons while also not messing up the presentation of the page
  3. Modify the page's JavaScript so that their newly substituted button does what the original pseudo-button did.

An overlay product cannot reliably do the above, because Step #1 requires privileged access to the browser. That's something a browser extension could do, but not something that can be done by a 3rd part script like an overlay.
